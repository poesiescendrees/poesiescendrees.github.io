Github, qu'est-ce que c'est ?

le site Github est un site web et un service de Cloud permettant de stocker, gérer et partager ses codes. Il est utilisé par un grand nombre de développeurs et informaticiens à travers le monde, souvent professionnellement. S'il recquiert des notions en programmation pour utiliser toutes ses fonctionnalités, vous pouvez tout-à-fait l'utiliser afin d'héberger vos codes, vos templates et pages HTML. Cela vous permet de les stocker tous à un endroit, évidemment, et de les partager s'ils sont en libre service, mais également de vous en servir comme site web statique basique (comme celui-ci par exemple) et hébergeur de CSS.
github desktop & éditeurs de code Comme énoncé plus haut, Github est utilisé dans le monde professionnel et son utilisation classique recquiert l'usage d'invites de commande et/ou du logiciel Github Desktop. Si vous ne voulez pas vous compliquer la vie avec ces derniers et utiliser Github simplement, il est possible de s'en passer pour les utilisations citées plus haut. Je vous explique comment faire ci-dessous.


1— Créer un compte


Commencez par vous créer un compte sur Github. Cochez les préférences que vous pensez préférables dans le panneau de personnalisation de votre compte pendant l'inscription.

Une fois inscrit·e, vous pouvez créer un repository via la toolbar.

3 — Créer un projet


Une fois inscrit·e, vous pouvez créer un repository via la toolbar.

Si vous souhaitez simplement stocker vos données et/ou héberger votre CSS pour le mettre en lien autre part vous pouvez créer le projet comme bon vous semble. Le répertoire général se comportera comme un dossier, dans lequel il est possible d'agencer les fichiers et autres dossiers comme sur votre ordinateur.

Si vous souhaitez utiliser un site statique, il faut donner un nom particulier à votre repository (qui sera le site en lui-même). Vous devez l'appeler votre-username.github.io, comme dans l'exemple ici où l'utilisateur s'appelle ghostlydaze et où le repository s'appelle donc ghostlydaze.github.io. Naturellement, vous ne pourrez donc créer qu'un seul site statique par compte Github.

5 — Stocker ses fichiers

général : créer des fichiers
Comme expliqué ci-dessus, le répertoire général se comporte comme un dossier. Vous pouvez donc créer le premier fichier du projet en allant sur la page principale de votre projet. Tout fichier peut être édité ou supprimé à tout moment.
Vous pouvez uploader un fichier depuis votre ordi ou le créer directement depuis Github. Pour cela, indiquez-y le nom du fichier et le contenu puis validez. Attention que le nom du fichier doit indiquer son type ! S'il s'agit d'une feuille de CSS, ça sera nom.css, si c'est un fichier HTML nom.html, si c'est un fichier de texte simple nom.text, etc.
Les fichiers peuvent être d'absolument tous types : des fichiers HTML, des fichiers texte, des feuilles CSS, mais également des images, du Javascript, des PDF,... Pour les PDF, images et autres fichiers qui ne peuvent pas être directement écrits sur un éditeur de texte, je vous conseille de les importer depuis votre ordi.
général : catégoriser en fichiers
À la création d'un fichier, ou en éditant un fichier existant, vous pouvez créer un dossier dans lequel le placer. Vous devez simplement indiquer comme titre de fichier créé : titre_dossier/titre_fichier en n'oubliant pas que le type du nom du fichier nom.css/html/etc. Dès que vous entrez la barre /, Github comprend que vous voulez créer un dossier et l'apparence du champ de texte changera. Visuel.

Une fois le fichier créé / les changements acceptés, vous pouvez retourner sur la page principale de votre projet et constater que le dossier a été créé. Vous pouvez dès à présent vous placer directement dans ce dernier pour y créer d'autres fichiers et d'autres dossiers.
héberger son CSSfacultatif
Vous pouvez héberger votre CSS sur Github et l'utiliser en lien externe à un code HTML. Cela n'a pas vraiment d'utilité pour les templates de FA puisque vous pouvez y gérer directement votre CSS. En revanche, vous pouvez vous en servir pour Tumblr ou un libre service de codes de fiches, de signatures etc. Notez qu'une fois que vous hébergez votre CSS, vous ne pouvez plus le modifier ! Donc je vous conseille peut-être de stocker dans votre CSS tout ce qui est inchangeable (la structure, le comportement général, etc) et de laisser tout ce qui l'est (les couleurs, les tailles de blocs, etc.) en-dehors.

Pour cela, créez votre fichier. Une fois sur la page qui lui est dédié, cliquez sur le bouton "Raw" qui créera la page CSS liée. Cette action n'est disponible qu'une fois par fichier (c'est pour cela que vous ne pourrez plus le modifier plus tard) donc soyez sûr·e que c'est bon, ou vous devrez recréer un autre fichier plus tard avec les modifications apportées.
Appuyer sur "Raw" vous renvoie automatiquement sur une page de CSS "brute". Le lien de cette page devrait se présenter comme cela : https://raw.githubusercontent.com/votre-pseudo/etc. Remplacez raw.githubusercontent par rawgit, de façon à avoir : https://rawgit.com/votre-pseudo/etc. Collez la nouvelle adresse URL dans le premier encadré de cette page. Cela vous produit un nouveau lien dans le second encadré qui est celui à utiliser pour lier votre CSS. Vous n'avez plus qu'à le coller dans le code ci-dessous et c'est bon !

L'utiliser fait en sorte que le CSS contenu dans la feuille de style liée s'applique à toute la page. (Faites donc attention à ce que vous y mettez, ici par exemple si vous avez suivi mon exemple j'ai défini un body{background: lightblue} mais si je l'utilise sur un message de forum, c'est tout le fond du body du sujet qui sera défini en lightblue !)
