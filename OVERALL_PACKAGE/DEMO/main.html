Bonjour ! Je vous présente ici un package de galeries CSS qui a pour but de vous aider à écrire vos codes de forums divers et variés afin que leur création soit plus facile & rapide, uniformisée donc plus facile d'entretien lors des changements de version & co. Pour se faire, le package comprend une succession de balises réparties en quatre galeries à combiner :

  • variables.css : cette galerie n'en est pas une à proprement parler, elle contient seulement les informations de base nécessaires aux autres galeries.
  • util.css : la galerie contient un certain nombre de balses utiles qui permettent de donner une structure quelconque à vos éléments. Par exemple, elle est fort utile dans la création de templates ou de fiches de présentation/lien/etc. afin de placer rapidement et avec très peu de code additionnel tous vos éléments comme vous le désirez.
  • style.css : la galerie offre un certain nombre de styles de police, blocs & co qui sont totalement customisables et ne devraient pas rentrer en désaccord avec le reste de vos balises CSS (à moins que vous en utilisiez certaines du même nom).
  • base_style.css : cette galerie, plus petite que les deux précédentes, donne aussi toutes sortes de styles de polices & et propriétés variées. Je l'ai séparée de la valerie style.css car contrairement à cette dernière, elle touche à des éléments de base utilisés presque partout. Elle est donc utile si vous commencez à coder une page ou un forum neuf mais vous n'en aurez certainement pas besoin pour une interface préexistante et déjà codée.

Les exemples repris en haut de page vous montrent un peu ce qu'il serait possible de réaliser à partir de ça. Le premier code factice de page de forum, par exemple, utilise les quatre galeries. Se faisant, le seul code additionnel nécessaire est celui-ci puisqu'il utilise les éléments styles de base et a "juste" besoin qu'on lui précise les largeurs exactes à donner à certains contenants. Le deuxième code factice de page de forum a un code HTML identique au premier exemple mais un CSS différent (le même que le 1er + des précisions pour modifier le style de base des blocs).
